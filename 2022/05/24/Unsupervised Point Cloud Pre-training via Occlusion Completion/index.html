Unsupervised Point Cloud Pre-training via Occlusion Completion

摘要

我们描述了一种简单的点云预训练方法。有三个步骤:1。在相机视图中遮挡所有点;2. 学习一个编码器-解码器模型来重建被遮挡点;3.使用编码器权重作为下游点云任务的初始化。我们发现，即使我们在单个数据集(ModelNet40)上进行预训练，这种方法也能在广泛的下游任务中提高跨不同数据集和编码器的准确性。具体来说，我们证明了我们的方法在对象分类、基于部分的和语义的分割任务方面优于以往的预训练方法。我们研究了预训练的特征，发现它们导致广泛的下游最小值，具有高的变换不变性，并具有与部分标签高度相关的激活

1 引言

对于目标检测[27,52,64]和分割[22,26,57,65]，已经出现了一系列令人兴奋的新点云模型。这些方法依赖于标记的大规模点云数据集。不幸的是，标记点云是一个挑战，原因有很多:(1)点云可能是稀疏的，闭塞的，低分辨率，使点的身份模糊;(2)非稀疏的数据集很容易达到上亿点(例如，用于对象分类的小密点云[63]和用于重建的大广点云[66]);(3)标记单独的点或绘制3D边框比标记2D图像[50]费时且容易出错。这些挑战阻碍了点云模型在标签数据稀缺的新现实环境中的部署。

然而，目前的3D传感模式(即3D扫描仪、立体相机、激光雷达)已经能够创建大型的未标记点云数据存储库[13,41]。这启发了最近的无监督预训练方法的研究，以学习点云模型初始化参数。最初的工作使用了潜在生成模型，如生成对抗网络(GANs)和自编码器。这些目前被自监督模型超越了。

受这一最近工作的启发，我们提出了Occlusion Completion遮挡补全(OcCo)，一种无监督的预训练方法，包括:(a)通过视点遮挡生成被遮挡点云的机制，(b)重建被遮挡点云的完成任务。遮挡+补全的思想基于三个观测:(1)一个预训练的模型，要准确地完成遮挡点云，需要了解这些点云的空间和语义属性。(2) 3D场景补全[45,9,19]已被证明是学习视觉定位[43]表示的一个有用的辅助任务。(3)基于掩码的完成任务已经成为自然语言处理中学习预训练表示的事实上的标准[11,32,36]，广泛用于图像[35]和图形[24]的预训练。

我们演示了在单个对象级数据集(ModelNet40)上的预训练可以提高一系列下游任务的性能，即使是在完全不同的数据集上。具体来说，我们发现OcCo与其他初始化技术相比具有以下特性:1)在少镜头学习实验中提高了样本效率;2)提高了对象分类、对象部分分割和语义分割的泛化能力;3)微调后发现更优的局部极小值;4)更多通过网络解剖描述的有意义的表示。5)抖动、平移和旋转变换下的聚类质量更好。

2 相关工作

由于在许多问题设置中取得了成功，例如自然语言理解[11,32]、对象检测[8,16]、图学习[23,24]和视觉定位[43]，无监督的预培训越来越受欢迎。目前，最常见的两种无监督的点云预训练方法是基于(i)生成建模和(ii)自我监督学习。生成式建模的工作包括基于生成式对抗网络(GANs)[54,1,14]、自动编码器[15,29,59]、归一化流[58]和近似凸分解[12]的模型。

然而，对点云进行无监督预训练的生成模型最近已经被自我监督方法超越[42,44,56]。这些方法通过学习预测跨数据集不变的点云的关键几何属性来工作。具体来说，[42]提出了一个基于重新排列排列的点云的训练前程序。它的工作原理是将点云分裂成k^3个体素，随机排列这些体素，然后训练一个模型来预测每个点的原始体素位置。其思想是，预先训练的模型通过学习这种重新排列隐含地学习点云的几何结构。然而，这一目标存在两个关键问题:体素表示不是排列不变的。因此，如果点云被旋转或平移，模型可以学习到非常不同的表示;2. 从真实物体和场景生成的点云与随机排列的云具有非常不同的结构，因此尚不清楚为什么在重新排列时准确的预先训练的权值将是对象分类或分割模型的良好初始化。另一个工作[44]使用覆盖树[6]来分层划分点，以实现少镜头学习。然后，他们训练一个模型将每个点分类到指定的分区。然而，由于覆盖树是为快速的最近邻居搜索而设计的，它们可能会任意地将点云(如飞机机翼、汽车轮胎)的语义上相邻的区域划分为层次结构的不同区域，从而忽略关键点云的几何形状。第三个作品，PointContrast[56]，使用对比学习来预训练场景点云的权重。他们的方法使用完整3D场景的不同视图之间已知的点向对应关系。这些逐点的通信需要对数据进行后处理，将不同的深度地图登记到单个3D场景中。因此，他们的方法只能应用于已注册的静态场景，限制了该方法的适用性:我们将OcCo和PointContrast的比较留到以后的工作中。在接下来的内容中，我们将展示基于简单的自我监督目标的无监督前训练:完成闭塞点云，产生的权重在下游任务中超过[42]和[44]。

补全3D形状来学习模型初始化并不是什么新鲜事，[43]使用场景补全[45,9,19]作为一项训练前任务来初始化用于视觉定位的3D体素描述符。为此，他们基于深度图像生成了部分体素化场景，并训练了一个可变自动编码器来完成。不同的是，我们的重点是描述一种学习点云模型初始化的技术。我们的目标是这种预先训练的初始化，以改善各种下游任务，包括在各种数据集上的少镜头学习，对象分类和分割。

OcCo的概述。1. 取任意点云数据集，并通过(a)随机采样一个摄像机视点，(b)从该视点移除隐藏点(对于所有实验，我们使用从ModelNet40生成的相同的遮挡数据集)，为每个输入生成遮挡对象;2. 训练编码器-解码器模型来完成被遮挡的点云(编码器可以是任何学习点云表示的模型，解码器可以是任何完成模型);3.使用学习到的编码器权重作为任何下游任务的初始化(例如，少镜头学习，对象分类，部分/语义分割)。我们表明，OcCo在多个模型和任务上优于各种训练前方法。

4.1 OcCo Pre-Training Setup

  • 数据集

对于所有实验，我们使用ModelNet40[55]作为训练前数据集。ModelNet40包含来自40个类别的12,311个合成CAD对象，数据集分为9,843/2,468个对象分别进行训练和测试。我们使用训练集构造一个训练前数据集。通过相机内部参数{f=1000， γ=0， ω=1600, h=1200}生成闭塞点云。对于每个点云，我们随机选择10个视点，其中偏摆、俯仰和滚动角度均匀地选择在0到2π之间，平移设为0。

4.2 Few-Tuning Setup

  • Few-shot learning

FSL目的是用少量的数据训练出泛化模型，常表示为“K-Way，N-Shot”，训练时，K个种类被随机选择，每个类别包含N个样本。我们比较了OcCo与Jigsaw[42]和cTree[44]，因为它优于以前的无监督方法[1,54,62,59]以及有监督变体[38,30,37,53]。我们遵循与cTree相同的设置，在ModelNet40和ScanObjectNN上评估之前，我们在ModelNet40上以“K-way N-shot”配置对模型进行预训练。
