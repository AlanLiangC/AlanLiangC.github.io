Back to Reality: Weakly-supervised 3D Object Detection with Shape-guided Label Enhancement

代码地址

摘要

在本文中，我们提出了一种弱监督的三维物体检测方法，通过位置级标注(即物体中心标注)，可以训练出强三维检测器。为了弥补损失的信息框注释中心,我们的方法,即回到现实(BR),利用合成3 d形状将疲软的标签转换成充分解释虚拟场景更强的监督,进而利用完美的虚拟标签来补充和完善真正的标签。具体来说，我们首先根据位置级标注提取的粗略场景布局，将三维形状组合成物理上合理的虚拟场景。然后，我们采用虚拟-真实域自适应方法回到现实中，细化弱标签，并利用虚拟场景监督检测器的训练。此外，我们提出了一个更具挑战性的基准，室内三维目标检测具有更多的对象大小的多样性，以更好的评估。只需不到5%的标注工作量，我们就可以在广泛使用的ScanNet数据集上实现与一些流行的完全监督方法相当的检测性能。

1 引言

尽管基于深度学习的点云目标检测取得了成功，但训练检测器仍需要大量的标记包围盒。这个问题严重限制了这些方法的应用，因为标记一个精确的3D框需要超过100s，即使是一个有经验的注释器[38]。因此，使用廉价标签的三维物体检测方法是值得实际应用的。出于这一点,增加已经注意weakly-supervised 3 d物体检测方法,可根据注释的形式分为两类:scene-level[35]和position-level(23、24),只有带注释的类标记对象中心和类分别为每一个对象。与标记一个边界框相比，这两种类型的注释对一个实例只需要不到1%和5%的时间，如表1所示。

Annotating time and detection results of different methods based on various types of annotation. The benchmark is detailed in Section 4. (BBox refers to box annotation. S-L and P-L mean scene-level and position-level annotations respectively.)

尽管单纯的场景标记更节省时间，但是难以学习单个目标的具体位置因为它缺乏位置信息。因此性能远非令人满意的[35]。考虑到时间精度的权衡，位置级注释是一个更实用的解决方案。然而，以往的位置级弱监督三维检测方法仍然需要大量精确标记的盒子，只能应对稀疏的户外场景[23,24]。对于复杂的室内探测任务，纯位置级弱监督方法仍在探索中。

BR的示范。我们将位置级标注作为场景的粗布局，利用它从三维形状库生成虚拟场景。在虚拟场景中应用物理约束来弥补从框标注到中心的信息丢失。在此基础上，提出了一种虚拟到真实域的自适应方法，将虚拟场景与真实场景结合起来监督真实场景的三维目标检测。虚线箭头表示训练监督

在本文中，我们提出了一种基于形状引导的标签增强方法，称为Back to Reality (BR)，用于弱监督三维目标检测1。为了降低人工成本，我们在三维空间中只标注每个物体的中心，并允许中心的标注误差。在很大程度上减少标注工作量的同时，从方框标注到中心标注的信息丢失是不可忽视的。为了解决这些问题，BR将弱标签转换为虚拟场景，虚拟场景中包含了大量丢失的信息，并反过来利用它们来监督真实场景的训练，如图1所示。我们的方法基于两个动机:

  1. 在3D视觉中，大规模的合成形状数据集是可用的。它们包含丰富的几何信息，可以为三维目标检测提供有力的先验辅助;

  2. 位置级标注不仅是对training的监督，还提供了场景的粗略布局。因此，我们根据粗略布局将三维形状组合成全标注的虚拟场景，并对其施加物理约束以弥补信息损失。然后提出一种虚拟到真实域的自适应方法，将检测器提取的全局特征和目标建议特征在真实场景和虚拟场景之间进行对齐。此外，该方法还可以利用虚拟场景中精确的中心标签来修正位置级标注的中心误差。通过这种方式，虚拟场景中包含的有用知识被转移回现实。在ScanNet[9]上的实验结果表明了该方法的有效性。

2 相关工作

2.1 3D Shape to Scane

由于大规模的合成3D形状数据集比真实场景数据集好获得的多，利用形状来辅助场景理解是一个很有前途的想法。现有的方法可以分为两类:有监督[4,5,8,42]和无监督[10,21,26,32,44]。对于监督方法，通常使用合成形状来完成不完美的真实场景扫描。给定一组CAD模型和一个真实的扫描，训练一个网络，预测如何将CAD模型放置在场景中，替换部分的、有噪声的真实物体[4,5,8,42]。训练过程中使用了人类注释的原始扫描对和对象对齐扫描对。由于监督方法需要额外的人力，这可能会限制三维形状数据集的充分利用。无监督方法通常用于数据扩充或数据集扩展。三维CAD模型按照基本的物理约束随机放置，生成混合的现实场景[10,44]或虚拟场景[21,26]。最近，RandomRooms[32]提出使用ShapeNet数据集对3D检测器进行无监督的预训练。我们的方法还利用3D形状以无监督的方式帮助对象检测。而在位置级弱监督检测任务中，我们的目标是利用合成形状增强弱标签，获得更强的监督。

2.2 3D Object Detection

早期的三维物体检测方法主要有基于模板的方法[18,20,25]和滑动窗口方法[39,40]。PointNet/PointNet++[29,30]的出现使基于深度学习的点云三维检测方法开始出现。然而，[6,7,17,28]中的方法依赖于生成2D提案，然后将其投射到3D空间中，这很难处理有严重遮挡的场景。最近，有人提出了直接消耗点云的网络[13,22,27,37,53]。尽管三维目标检测方法发展迅速，但由于标记数据有限，其应用仍受到一定的限制。为了减少人工标注的工作量，最近提出了弱监督方法[23,24,31,35]、半监督方法[43,51]和无监督的预训练方法[14,32,47,49]。然而，预训练方法依赖于巨大的计算资源，以对比学习的方式训练网络。半监督方法遵循与二维[41]相似的程序，并没有充分探索3d数据的特征。因此，我们研究针对三维目标检测任务的弱监督方法。

3 方法

我们BR方法的框架。给出带有位置级注释的真实场景，我们首先增强弱标签以获得完整注释的虚拟场景。然后将真实场景和虚拟场景送入检测器，分别用弱监督和全监督检测损失进行训练。在训练过程中，我们利用虚拟场景中的精确对象中心来细化真实场景中的不精确对象中心。采用强弱对抗域适应方法对两个域的特征分布进行对齐。全局鉴别器输出对每个场景的判断，建议鉴别器输出对每个对象建议的判断。(此处GRL为梯度反转层;Dg、Dp分别代表global、proposal discriminator。)

图2说明了我们的方法的框架。给出带有位置级标注的真实场景，我们利用3D形状将弱标签转换为虚拟场景，并利用虚拟场景为检测器的训练提供额外的监督。在本节中，我们首先讨论弱监督设置，然后演示BR的关键步骤。

3.1 Position-level Annotation

选择在3 d空间是困难的,我们将标记过程分为两个步骤:首先我们标签的中心一个对象在一个适当的场景的2 d视图,并计算出直线通过这个中心和焦点相机根据相机参数的二维视图。其次，我们在直线上选择一个点来确定物体在三维空间中的中心。该策略对实例进行标记的时间小于5s，标记误差控制在实例大小的10%以内。在对三维场景进行扫描时，很多情况下都可以获得网格数据。我们假设网格在输入中是可用的。然而，在我们的方法和实验中也考虑了我们只有点云数据的情况。

3.2 Shape-guided Label Enhancement

位置级标注所需标注时间少得多，但信息丢失严重，主要表现在两个方面:1)物体尺寸信息丢失;2)对象中心不精确。尽管如此，位置级注释可以提供场景的粗略布局。通过根据布局组装合成的3D形状，我们能够增强弱标签，并生成精确标注的虚拟场景，其中大小可用，中心精确。我们的标签增强方法分为两步:1)首先计算三维形状的基本属性;2)然后我们放置这些形状，根据标签生成物理上合理的虚拟场景。我们在附录3中提供了一些实现细节。

  • Definition of Shape Properties

给定一个点云，我们假设它是轴对齐和标准化成一个单位球，O的长度，宽度和高度定义为，然后我们把形状的类别分为三类:支撑者、站者和被支撑者。前两者都是只能靠地面支撑的物体，不同的是站着的人不太可能支撑其他东西。其他类别是受支持的。

如果一个形状属于Supporter，则计算他的三个属性：最小围矩形面积（MER），支撑表面高度（SSH），和支撑面紧实度（CSS），M ER是在XY平面内计算的，它是形状中所有点的最小矩形。SSH是其他对象可以站立的最高表面的高度。CSS是一个布尔值，表示支撑面是否可以用MER近似

  • Virtual Scene Generation

我们采用了三阶段的方法来构建虚拟场景，相当于一步一步地生成每个形状的位置:

1)我们首先细化由位置级标注提供的粗布局并生成初始位置;

2)通过恢复物体之间的支持关系生成重力感知位置;

3)最后，我们生成碰撞感知位置，使虚拟场景在物理上合理。管道如图3所示。

介绍了我们的三阶段流水线虚拟场景生成方法。我们首先从网格数据中提取水平段，并用它们来细化由位置级标注提供的粗布局。然后根据新的布局将合成的三维形状放置到虚拟场景中，构建初始虚拟场景。然后我们在虚拟场景中应用重力和碰撞约束来恢复失去的物体之间的物理关系，使场景更加逼真。

为了生成初始位置，我们需要从场景的几何信息中恢复出更精确的布局。给定一个mesh格式的场景，我们首先使用基于法向的图切割方法对网格进行过分割[11,15]。结果是一个分割图，其中节点表示分割，边表示邻接关系。对于水平的，面积∗大于Amin，高度∗大于Hmin的*，我们迭代地合并它们的邻居如果水平段与相邻段的高度差小于∆h。合并后的线段视为一个整体，并将合并后的线段设置为与原水平线段相同的高度。合并后，每个水平段由其M ER表示。如果只有一个支持者的中心落在一个M ER中，我们将这个M ER分配给该支持者。当多个支持者的中心落在同一个M ER时，我们根据这些中心对水平段进行K-means聚类，并分别计算每个支持者的M ER

实例索引是一个从1到场景中对象数量的整数，是中心的坐标，是在三个轴上的尺度，是旋转角度，M和H代表supporter的MER和SSH。当S为假时，它们被设置为None。如果形状被分配了一个水平段，我们使用该段的M ER来初始化上述参数。也就是说，我们选择一个CSS为True的支持者，让这个支持者的M ER与水平段重叠。否则我们进行随机初始化。如果只有点云数据可用，我们只需执行随机初始化，以下阶段是相同的。

接下来，我们遍历初始位置以生成重力感知位置。在这个过程中，我们只需要改变位置字典中的z和SSH。对于支持者和站立者，我们直接将他们的底部与地面对齐(即XY平面)。对于一个支撑物，如果它的(x, y)落在任何支撑物的M ER中，我们将它分配给最近的支撑物，并将其底部与支撑物表面对齐。

之后，我们移动形状，以获得碰撞感知的位置。这个阶段只有位置字典中的x和y会被改变。首先，我们移动地面上的物体，那些有支撑的物体，如果有的话，它们会一起移动。然后对于每个支持者，我们移动它的支持者，直到没有重叠。需要注意的是，三个生成阶段不仅可以使虚拟场景更加逼真，还可以削弱中心标签不精确的影响。因此，虚拟场景生成方法对标记误差具有鲁棒性。

3.4 Virtual2Real Domain Adaptation

虽然标签增强方法能够生成物理上合理的全注释虚拟场景，但它们与真实场景之间仍然存在巨大的域差距(例如，在虚拟场景中缺少背景，如墙壁)。因此，我们需要在完美的虚拟标签中挖掘有用的知识来弥补位置级标注的信息损失，而不是仅仅依靠虚拟场景。
