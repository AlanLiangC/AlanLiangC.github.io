Helios++仿真流程及文件架构

1 仿真指令

  • run/helios.exe [survey-file]

该软件有一个模块化的架构，其中的模拟信息(即平台、场景和扫描仪)是从XML文件和链接的文件中读取的。[survey-file]是一个XML文件的绝对或相对路径。在这个XML文件中，提供了指向场景XML、平台XML和扫描程序XML的路径。数据目录中提供了示例场景、平台和扫描仪。

  • 例如
  • run/helios.exe data/surveys/demo/tls_arbaro_demo.xml

2 XML文件结构

  • 在survey的xml中，可以配置扫描位置或路点，并控制全波形设置、平台设置和扫描仪设置。以/demo/tls_arbaro_demo.xml为例
  • Scene XML：场景XML文件链接到不同的几何部分，可以是不同类型的不同类型，如多边形网格，地理参考栅格（地理仪），点云或体素。可以传递不同的参数和过滤器以定义如何加载和预处理链接的文件，包括转换，例旋转和单个场景部件的缩放。
  • Platform XML：平台XML包含不同平台的定义，例如用于地面激光扫描模拟的固定三脚架、用于移动激光扫描模拟的车辆和用于机载激光扫描模拟的飞机。许多平台是预定义的(三脚架，四轴飞行器，Cirrus SR-22，…)，并可以扩展。
  • scanner XML：扫描器XML包含扫描器规范，如光束发散度、精度、偏转器类型、脉冲频率或脉冲长度。在数据文件夹中的scaners_als .xml和scaners_tls .xml中预定义了许多扫描仪。

3 Survey

3.1 Survey definition

Survey.xml 文件包含对构建模拟所需组件的引用: 场景、平台和扫描仪。它还包含定义扫描位置或轨迹所需的路点信息。下面将详解有关Survey.xml中所有可能的标记和参数。

3.2 Scanner settings and platform settings

Scanner设置和platform设置是在 < survey >-tag 和/或 < leg >-tags 之前的文档级别定义的。在文件的开头设置一个或多个带有定义的扫描仪设置的模板，可以为不同的分支重用这些设置。在每个 < leg > 的 < scannersettings > 或 < platformsettings > 中，可以使用模板参数通过 ID 引用全局定义的设置。此外，所有设置都可以显式地在 < scannersettings > 和 < leg > 的 platformSettings 中给出。如果在模板中和使用模板的分支中定义了相同的设置，则采用分支中给出的值。所以优先级总是: Leg-> Template-> Default

3.3 Detector settings

在 detectorsettings标签中，可以指定探测器的精度、最小范围和最大范围。检测器设置是在Survey.xml中定义的。

3.4 Fullwave settings

FWFSettings 用于在空间和时间上配置离散化的完整波形，以及用于峰值检测的窗口大小。FWFSettings 在Survey.xml中定义。

3.5 Leg definition

对于每个扫描位置或路点，定义一个 <leg>。可选地，stripId 可以被分配到一个leg。将对具有相同 stripId 的leg进行分组，并将其模拟的点云写入单个输出文件，该文件由 stripId 命名。没有 stripId 的分支被视为独立的条带，其输出连续编号(leg000、 leg001等)。在 <leg> 标签中，平台位置在<platformsettings>标签中提供。对于动态平台，至少需要定义两个对应于起止点的路径，并且两个路径点之间的速度通过参数 movePerSec _ m 来设置。移动的方向是由下一个航路点的位置决定的。地面参数对陆地测量是有用的。默认情况下是“ false”，但当设置为“ true”时，平台将自动放置在地面上，而不管指定的 z 坐标。地面被确定为在腿的 xy 位置数据中的最低 z 坐标。具体到直升机平台，转弯模式可以是特定的(smoothTurn 或 stopAndTurn) ，这在平台中进行了解释和可视化。

每条腿的扫描仪设置都在 < scannersettings >标签中定义。这包括扫描仪是否启动、脉冲频率、扫描角度、扫描频率、头部旋转和轨迹输出。它们也可以从模板加载，模板是在 < document > 开头的 < survey > 之前定义的。这些设置在Scanner.xml设置中有说明。

重要提示: 多功能平台存在一些 bug，比如当使用 slowdownEnabled或者用户定义的速度(movePerSec _ m)不被使用时，legs 不会结束。我们建议使用线性路径平台代替。在 platforms.xml 中，可能是这样的:

3.6 XML Tag and Parameter Summary

Output

输出位置

  • Helios + + 支持多种输出格式的点云，轨迹数据和全波数据。

  • 输出点云和(可选)的完整波形和轨迹文件存储在

[cwd]/output/Survey Playback/[name of survey as defined in survey.xml]/[time stamp of simulation start]/points/[number of scan position].[xyz|las]

比如

helios-plusplus/output/Survey Playback/TLS Arbaro/2016-09-28_15-17-52/points/leg000_points.xyz

ASCII point format

默认情况下，每个分支的点云被编写为以下格式分隔的 ASCII XYZ 文件(. XYZ) :

  • intensity：强度

  • echoWidth：只有在设置了命令行属性 calcachochwidth 时才计算回波宽度，否则所有点的回波宽度都将设置为0.000。

  • returnNumber：回波索引

  • numberOfReturns：回波数目

  • fullwaveIndex：一个索引，对应于 fullwave.txt 输出文件中的行索引。它与脉冲 ID 相同，因此来自同一脉冲具有相同的fullwaveIndex

  • hitObjectId：场景中各个对象的 ID。Id 以0开头，对应场景中存储的对象的顺序。

  • class：取自原始材料，这是通过材料 MTL 文件中的helios_classification字段指定的。

  • gpsTime： 将点与轨迹关联起来。它表示在 GPS 周开始(周日上午0:00开始)后，脉冲发出的时间戳(秒) + 300,000。模拟开始日期取自系统时钟，然后继续使用模拟时间。

LAS point format

使用参数 -- lasOutput，可以将点云写入 ASPRS LAS 1.4格式(Point Data Record Format 6;.las)。而不是 ASCII。Echo _ width、 fullwaveIndex 和 hitObjectId 作为额外字节存储，其他字段分别存储在 LAS Format 的各个字段中。

点云的坐标移动到场景的包围盒的左下角，并应用缩放。默认情况下，缩放比例为0.0001(相当于1/10毫米分辨率) ，但可以通过在命令行上使用 -- lasScale 参数进行更改。

ASCII trajectory format

如果在Survey.xml的 scannerSettings 标签中的定义了trajectoryTimeInterval_s，那么轨迹就会以这种格式写入:

X Y Z gpsTime roll pitch yaw

gpsTime 以 GPS 周开始后的秒数 + 300,000为单位，对应于点云输出中的 gpsTime。偏航轴周围的角度是以度数提供的，其位置和姿态与平台在全球参考框架中的原点和轴线有关，并不一定显示扫描仪的位置和方向。在测量或平台 XML 中，这个位置/方向可以通过提供杠杆臂和错位作为安装参数来改变
